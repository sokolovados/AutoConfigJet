Для заполнения шаблонов:

  • IP

    используется в формате ip.option ( например ip.ip_mask )
    • ip: 192.168.1.2
    • ip_mask: 192.168.1.2/24
    • network: 255.255.255.0
    • def_gateway_last: 192.168.1.254
    • def_gateway_first: 192.168.1.1

  • trunk_vlans

    имеет трехслойную структуру. Используется в формате trunk_vlans.option.vendor_option.
    • Option :
      1. all_vlans : включает в себя данные форм User,Mgmt,Trunk vlan
      2. vlans_without_mgmt :включает в себя данные форм User,Trunk vlan
      3. vlans_without_mgmt_user :включает в себя данные форм Trunk vlan
    • Vendor option :
      1. dlink : возвращает запись вида '1-3,4,5' в формате ['1', '2', '3', '4', '5']
      2. fiberhome :возвращает запись вида '1-3,4,5' в формате ['1-3', '4', '5']
      3. huawei :возвращает запись вида '1-3,4,5' в формате ['1 to 3', '4', '5']

  • hostname

  • snmp_location

  • user_vlan

  • mgmt_vlan

  • mvlan

  • op_description

  • op_checkbox :

    содержит значение чекбокса (True или False). Применяется в if условиях.

Интерфейс админ-панели:

Вкладки админ панели представляют собой графическую оболочку управления таблицами БД конфигуратора и директориями приложения.

таблицы:

  • Regions:

    агрегирует данные остальных таблиц. Используем при добавлении нового региона.

  • Users:

    данные для авторизации пользователей. т.к доменная авториация не работает, при необходимости добавляем имя пользователя и пароль (oas)

  • Models:

    зависимости формата vendor:model.

  • Recommendations:

    таблица с подсказками , которые опционально можно добавлять на html страницу через влкадку Region

  • Accounting:

    Учет сгенерированных конфигов. Содержит информацию и заполненных полях, пользователе, времени, номере сущности Remedy. Работет поиск.

директории:

  • Config Files

    Готовые конфиги , при необходимости можно найти нужный.

  • Template Files

    Директории с шаблонами. Есть возможность редактирования/загрузки/удаления шаблонов. БУДЬТЕ ВНИМАТЕЛЬНЫ!
